Comments

Single line comment

Single line comments start with // and can be found at any position in the line. 
 // a standalone single line comment
 println("hello"); // a comment till the end of the line

Multiline comment

A multiline comment starts with /* and can be found at any position in the line. The characters following /* will be considered part of the comment, including new line characters, up to the first */ closing the comment. 
  /* a standalone multiline comment
  spanning two lines */
  println("hello"); /* a multiline comment starting
                   at the end of a statement */
  println(1 /* one */ + 2 /* two */);

Identifiers

Identifiers start with a letter, a dollar or an underscore. They cannot start with a number.
A letter can be in the following ranges:
  • 'a' to 'z' (lowercase ascii letter)
  • 'A' to 'Z' (uppercase ascii letter)
Then following characters can contain letters and numbers.
Here are a few examples of valid identifiers (here, variable names): 
  name
  item1
  _startUnderscore
  with_underscore
  $startDollar
But the following ones are invalid identifiers: 
  1name
  a+b
  #abc
All keywords are not valid identifiers.

Literals

A literal is the source code representation of a value.

Strings

String literals represent sequence of characters enclosed in double quotes. 
 a = "Hello";
 b = "a";
 c = "";

Numbers

Number literals represent sequence of numbers. 
 a = 123;
 b = 3.14;
 b = -0.0012;

Booleans

Boolean literals represent two values: true and false. 
 a = true;
 b = false;

Lists

List literals represent a comma-separated list of values, surrounded by square brackets: 
 a = [1, 2, 3];
 b = [1, "abc", -20.56, true];

Maps

Maps literals represent a comma-separated list of key/value pairs, surrounded by curly brackets. Maps associate keys to values, separating keys and values with colons: 
 a = {"red": "#FF0000", "green": "#00FF00", "blue": "#0000FF"};
 b = {1: "one", 2: "two"};	
 c = {"name": "Alex", "amount": 5000.00, "paid": true};
